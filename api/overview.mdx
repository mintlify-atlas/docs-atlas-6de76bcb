## Introduction

The MiroFish API provides a comprehensive REST interface for building swarm intelligence prediction engines. The API enables you to:

- Build knowledge graphs from documents
- Generate ontologies and entity relationships
- Create and run agent-based simulations
- Generate analytical reports and insights
- Interview simulation agents

## Base URL

For local deployment:

```
http://localhost:5001
```

## API Architecture

MiroFish uses a RESTful architecture with JSON request/response formats. The API is organized into three main modules:

### Graph API (`/api/graph/*`)
Manage projects, generate ontologies, and build knowledge graphs from documents.

### Simulation API (`/api/simulation/*`)
Create simulations, generate agent profiles, run simulations, and interview agents.

### Report API (`/api/report/*`)
Generate analytical reports, check report status, and chat with the Report Agent.

## Response Format

All API responses follow a consistent JSON structure:

```json
{
  "success": true,
  "data": {
    // Response data
  }
}
```

Error responses include an `error` field:

```json
{
  "success": false,
  "error": "Error description",
  "traceback": "Detailed error trace (in development)"
}
```

## Error Handling

### HTTP Status Codes

<ResponseField name="200" type="OK">
  Request succeeded
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid request parameters or missing required fields
</ResponseField>

<ResponseField name="404" type="Not Found">
  Resource not found (project, simulation, report, etc.)
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Server error during processing
</ResponseField>

### Common Error Responses

<CodeGroup>



</CodeGroup>

## Quick Start

Here's a complete workflow example:

<CodeGroup>
```bash Step 1: Generate Ontology
curl -X POST http://localhost:5001/api/graph/ontology/generate \
  -F "files=@document.pdf" \
  -F "simulation_requirement=Analyze sentiment about university policy" \
  -F "project_name=University Sentiment Analysis"
```

```bash Step 2: Build Knowledge Graph
curl -X POST http://localhost:5001/api/graph/build \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_abc123",
    "chunk_size": 500,
    "chunk_overlap": 50
  }'
```

```bash Step 3: Create Simulation
curl -X POST http://localhost:5001/api/simulation/create \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_abc123",
    "enable_twitter": true,
    "enable_reddit": true
  }'
```

```bash Step 4: Run Simulation
curl -X POST http://localhost:5001/api/simulation/run \
  -H "Content-Type: application/json" \
  -d '{
    "simulation_id": "sim_xyz789",
    "platform": "parallel",
    "max_rounds": 100
  }'
```
</CodeGroup>

## Asynchronous Operations

Many operations (graph building, simulation preparation, report generation) are asynchronous and return a `task_id`:

```json
{
  "success": true,
  "data": {
    "task_id": "task_123abc",
    "message": "Task started"
  }
}
```

Use the task status endpoints to monitor progress:

```bash
curl http://localhost:5001/api/graph/task/task_123abc
```
