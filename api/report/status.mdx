## Get Report Generation Status

Check the status of a report generation task.

```bash
POST /api/report/generate/status
```

### Request Body (JSON)

<ParamField body="task_id" type="string">
  Task ID from the generate endpoint (for active tasks)
</ParamField>

<ParamField body="simulation_id" type="string">
  Simulation ID to check if report is complete
</ParamField>

<Note>
  Provide either `task_id` (for in-progress generation) or `simulation_id` (to check for completed reports).
</Note>

### Response

<ResponseField name="success" type="boolean">
  Whether the request succeeded
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Generation status">
    <ResponseField name="task_id" type="string">
      Task identifier (if querying active task)
    </ResponseField>
    <ResponseField name="simulation_id" type="string">
      Simulation identifier
    </ResponseField>
    <ResponseField name="report_id" type="string">
      Report identifier (when completed)
    </ResponseField>
    <ResponseField name="status" type="string">
      Status: `processing`, `completed`, `failed`
    </ResponseField>
    <ResponseField name="progress" type="number">
      Completion percentage (0-100)
    </ResponseField>
    <ResponseField name="message" type="string">
      Current operation description
    </ResponseField>
    <ResponseField name="already_completed" type="boolean">
      Whether report was already generated
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>
```bash cURL - By Task ID
curl -X POST http://localhost:5001/api/report/generate/status \
  -H "Content-Type: application/json" \
  -d '{"task_id": "task_report456"}'
```

```bash cURL - By Simulation ID
curl -X POST http://localhost:5001/api/report/generate/status \
  -H "Content-Type: application/json" \
  -d '{"simulation_id": "sim_xyz789"}'
```

```javascript JavaScript
const response = await fetch(
  'http://localhost:5001/api/report/generate/status',
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      task_id: 'task_report456'
    })
  }
);
const data = await response.json();
```
</CodeGroup>

## Get Report Details

Retrieve a complete generated report.

```bash
GET /api/report/{report_id}
```

### Path Parameters

<ParamField path="report_id" type="string" required>
  Report identifier (e.g., `report_abc123`)
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the request succeeded
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Report object">
    <ResponseField name="report_id" type="string">
      Report identifier
    </ResponseField>
    <ResponseField name="simulation_id" type="string">
      Associated simulation ID
    </ResponseField>
    <ResponseField name="status" type="string">
      Report status: `generating`, `completed`, `failed`
    </ResponseField>
    <ResponseField name="outline" type="object">
      Report outline/structure
    </ResponseField>
    <ResponseField name="markdown_content" type="string">
      Full report content in Markdown format

## Polling Strategy

For optimal UX, use progressive polling intervals:

```javascript
async function pollReportWithBackoff(taskId) {
  let interval = 2000; // Start with 2 seconds
  const maxInterval = 10000; // Max 10 seconds
  
  while (true) {
    const response = await fetch(
      'http://localhost:5001/api/report/generate/status',
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ task_id: taskId })
      }
    );
    const data = await response.json();
    
    if (data.data.status === 'completed' || data.data.status === 'failed') {
      return data;
    }
    
    await new Promise(resolve => setTimeout(resolve, interval));
    
    // Increase interval (exponential backoff)
    interval = Math.min(interval * 1.5, maxInterval);
  }
}
```
