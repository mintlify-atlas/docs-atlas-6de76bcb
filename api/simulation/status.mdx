## Get Simulation Status

Retrieve the current status and details of a simulation.

```bash
GET /api/simulation/{simulation_id}
```

### Path Parameters

<ParamField path="simulation_id" type="string" required>
  Simulation identifier (e.g., `sim_xyz789`)
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the request succeeded
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Simulation state">
    <ResponseField name="simulation_id" type="string">
      Simulation identifier
    </ResponseField>
    <ResponseField name="project_id" type="string">
      Associated project ID
    </ResponseField>
    <ResponseField name="graph_id" type="string">
      Associated knowledge graph ID
    </ResponseField>
    <ResponseField name="status" type="string">
      Current status: `created`, `preparing`, `ready`, `running`, `completed`, `stopped`, `failed`
    </ResponseField>
    <ResponseField name="entities_count" type="number">
      Number of entities/agents
    </ResponseField>
    <ResponseField name="entity_types" type="array">
      Types of entities included
    </ResponseField>
    <ResponseField name="enable_twitter" type="boolean">
      Whether Twitter platform is enabled
    </ResponseField>
    <ResponseField name="enable_reddit" type="boolean">
      Whether Reddit platform is enabled
    </ResponseField>
    <ResponseField name="config_generated" type="boolean">
      Whether simulation configuration has been generated
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 creation timestamp
    </ResponseField>
    <ResponseField name="updated_at" type="string">
      ISO 8601 last update timestamp
    </ResponseField>
    <ResponseField name="error" type="string">
      Error message if status is `failed`
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl http://localhost:5001/api/simulation/sim_xyz789
```

```javascript JavaScript
const response = await fetch(
  'http://localhost:5001/api/simulation/sim_xyz789'
);
const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'http://localhost:5001/api/simulation/sim_xyz789'
)
data = response.json()
```
</CodeGroup>

## Get Preparation Status

Check the status of simulation preparation (profile generation).

```bash
POST /api/simulation/prepare/status
```

### Request Body (JSON)

<ParamField body="task_id" type="string">
  Task ID from the prepare endpoint (for in-progress tasks)
</ParamField>

<ParamField body="simulation_id" type="string">
  Simulation ID (to check if preparation is complete)
</ParamField>

<Note>
  Provide either `task_id` (for active tasks) or `simulation_id` (to check completion status).
</Note>

### Response

<ResponseField name="success" type="boolean">
  Whether the request succeeded
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Preparation status">
    <ResponseField name="task_id" type="string">
      Task identifier (if querying active task)
    </ResponseField>
    <ResponseField name="simulation_id" type="string">
      Simulation identifier
    </ResponseField>
    <ResponseField name="status" type="string">
      Status: `not_started`, `processing`, `completed`, `ready`, `failed`
    </ResponseField>
    <ResponseField name="progress" type="number">
      Completion percentage (0-100)
    </ResponseField>
    <ResponseField name="message" type="string">
      Current operation description
    </ResponseField>
    <ResponseField name="progress_detail" type="object">
      Detailed progress information
      <Expandable title="Progress details">
        <ResponseField name="current_stage" type="string">
          Current stage: `reading`, `generating_profiles`, `generating_config`, `copying_scripts`
        </ResponseField>
        <ResponseField name="current_stage_name" type="string">
          Human-readable stage name
        </ResponseField>
        <ResponseField name="stage_index" type="number">
          Current stage number
        </ResponseField>
        <ResponseField name="total_stages" type="number">
          Total number of stages
        </ResponseField>
        <ResponseField name="stage_progress" type="number">
          Progress within current stage (0-100)
        </ResponseField>
        <ResponseField name="current_item" type="number">
          Current item being processed
        </ResponseField>
        <ResponseField name="total_items" type="number">
          Total items in current stage
        </ResponseField>
        <ResponseField name="item_description" type="string">
          Description of current item
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="already_prepared" type="boolean">
      Whether preparation was already completed
    </ResponseField>
    <ResponseField name="prepare_info" type="object">
      Information about completed preparation (when already_prepared=true)
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>
```bash cURL - By Task ID
curl -X POST http://localhost:5001/api/simulation/prepare/status \
  -H "Content-Type: application/json" \
  -d '{"task_id": "task_prep123"}'
```

```bash cURL - By Simulation ID
curl -X POST http://localhost:5001/api/simulation/prepare/status \
  -H "Content-Type: application/json" \
  -d '{"simulation_id": "sim_xyz789"}'
```

```javascript JavaScript
const response = await fetch(
  'http://localhost:5001/api/simulation/prepare/status',
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      task_id: 'task_prep123'
    })
  }
);
const data = await response.json();
```
</CodeGroup>

## Get Run Status

Check the runtime status of a simulation.

```bash
GET /api/simulation/status/{simulation_id}
```

### Path Parameters

<ParamField path="simulation_id" type="string" required>
  Simulation identifier
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the request succeeded
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Run state">
    <ResponseField name="simulation_id" type="string">
      Simulation identifier
    </ResponseField>
    <ResponseField name="runner_status" type="string">
      Runner status: `idle`, `running`, `completed`, `stopped`, `failed`
    </ResponseField>
    <ResponseField name="current_round" type="number">
      Current simulation round
    </ResponseField>
    <ResponseField name="total_rounds" type="number">
      Total rounds (from config or max_rounds parameter)
    </ResponseField>
    <ResponseField name="twitter_status" type="string">
      Twitter simulation status
    </ResponseField>
    <ResponseField name="reddit_status" type="string">
      Reddit simulation status
    </ResponseField>
    <ResponseField name="started_at" type="string">
      ISO 8601 timestamp when simulation started
    </ResponseField>
    <ResponseField name="completed_at" type="string">
      ISO 8601 timestamp when simulation completed (if applicable)
    </ResponseField>
    <ResponseField name="process_pid" type="number">
      Process ID of runner
    </ResponseField>
    <ResponseField name="error" type="string">
      Error message (if failed)
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl http://localhost:5001/api/simulation/status/sim_xyz789
```

### Response Example

```json
{
  "success": true,
  "data": {
    "simulation_id": "sim_xyz789",
    "runner_status": "running",
    "current_round": 47,
    "total_rounds": 100,
    "twitter_status": "running",
    "reddit_status": "running",
    "started_at": "2025-03-13T11:00:00Z",
    "process_pid": 12345
  }
}
```

## Simulation Status Values

### Lifecycle States

<ResponseField name="created" type="string">
  Simulation created, needs preparation
</ResponseField>

<ResponseField name="preparing" type="string">
  Generating agent profiles and configuration
</ResponseField>

<ResponseField name="ready" type="string">
  Prepared and ready to run
</ResponseField>

<ResponseField name="running" type="string">
  Currently executing simulation
</ResponseField>

<ResponseField name="completed" type="string">
  Simulation finished successfully
</ResponseField>

<ResponseField name="stopped" type="string">
  Manually stopped by user
</ResponseField>

<ResponseField name="failed" type="string">
  Simulation failed with error
</ResponseField>

## Polling for Progress

For long-running operations, poll status endpoints:

```javascript
async function pollPreparationStatus(taskId) {
  while (true) {
    const response = await fetch(
      'http://localhost:5001/api/simulation/prepare/status',
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ task_id: taskId })
      }
    );
    const data = await response.json();
    
    console.log(`Progress: ${data.data.progress}% - ${data.data.message}`);
    
    if (data.data.status === 'completed' || data.data.status === 'ready') {
      console.log('Preparation complete!');
      break;
    }
    
    if (data.data.status === 'failed') {
      console.error('Preparation failed:', data.data.error);
      break;
    }
    
    await new Promise(resolve => setTimeout(resolve, 2000)); // Wait 2 seconds
  }
}
```
