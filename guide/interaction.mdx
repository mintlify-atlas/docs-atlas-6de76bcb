## Overview

MiroFish provides powerful interaction capabilities to help you understand simulation results and agent behaviors. You can interview individual agents, query the Report Agent for insights, and explore the knowledge graph interactively.

## Interview Individual Agents

One of MiroFish's unique features is the ability to "interview" agents about their simulated behaviors, motivations, and perspectives.

### How Agent Interviews Work

<Steps>
  <Step title="Agent Context Loading">
    The agent's full persona, memory, and action history are loaded
  </Step>
  
  <Step title="Question Processing">
    Your question is presented to the agent's LLM-powered persona
  </Step>
  
  <Step title="In-Character Response">
    The agent responds based on their profile, experiences, and simulated worldview
  </Step>
</Steps>

<Info>
Agents respond **in character**, reflecting their persona, biases, and simulated experiences during the simulation.
</Info>

### Starting an Interview

```bash
POST /api/simulation/{simulation_id}/interview
Content-Type: application/json

{
  "agent_id": 23,
  "message": "Why did you support the new AI policy?",
  "chat_history": []
}
```

### Parameters

<ParamField path="agent_id" type="integer" required>
  The ID of the agent to interview (from simulation profiles)
</ParamField>

<ParamField path="message" type="string" required>
  Your question or prompt to the agent
</ParamField>

<ParamField path="chat_history" type="array">
  Previous conversation history for multi-turn interviews
  
  ```json
  [
    {"role": "user", "content": "What's your opinion on..."},
    {"role": "assistant", "content": "As a professor, I believe..."}
  ]
  ```
</ParamField>

### Response

```json
{
  "success": true,
  "data": {
    "agent_id": 23,
    "agent_name": "Dr. Sarah Chen",
    "response": "As a professor who's spent years in academia, I supported the policy because it addresses a critical need for AI literacy among students. My research in AI ethics made me particularly aware of the risks of unregulated AI use in education. During the simulation, I witnessed concerning instances of students relying too heavily on AI without understanding its limitations. The policy provides necessary guardrails while still allowing innovation.",
    "tool_calls": [],
    "timestamp": "2026-03-13T16:15:00"
  }
}
```

### Multi-Turn Conversations

Build on previous responses for deeper insights:

```bash
POST /api/simulation/{simulation_id}/interview
{
  "agent_id": 23,
  "message": "Can you elaborate on the 'concerning instances' you mentioned?",
  "chat_history": [
    {
      "role": "user", 
      "content": "Why did you support the new AI policy?"
    },
    {
      "role": "assistant",
      "content": "As a professor who's spent years in academia..."
    }
  ]
}
```

<Tip>
Maintain conversation context by including full chat history for follow-up questions. This enables natural, flowing interviews.
</Tip>

## Query the Report Agent

The Report Agent can answer questions about simulation results using its analysis tools.

### Interactive Q&A

```bash
POST /api/report/chat
Content-Type: application/json

{
  "simulation_id": "sim_m1n2o3p4",
  "message": "What were the main concerns raised by students?",
  "chat_history": []
}
```

### Report Agent Capabilities

Unlike agent interviews, the Report Agent:

  <Card title="Uses Tools" icon="wrench">
    Autonomously calls analysis functions:
    - Graph search
    - Statistical queries
    - Agent interviews
    - Insight generation
  </Card>
  
  <Card title="Objective Analysis" icon="microscope">
    Provides analytical perspective:
    - Pattern identification
    - Data synthesis
    - Multi-source insights
    - Trend analysis
  </Card>
</CardGroup>

        "results_count": 34
      },
      {
        "tool": "panorama",
        "entity_type": "Student",
        "stats": {"total": 45, "active": 42}
      }
    ],
    "sources": [
      "Knowledge graph: 34 relevant facts",
      "Simulation actions: 247 student posts analyzed",
      "Agent interviews: 3 representative students"

### Useful Report Agent Queries

**Quantitative Questions:**
- "How many agents participated in the discussion?"
- "What percentage of posts were supportive vs. critical?"
- "Which agent had the most influence?"

**Qualitative Questions:**
- "What themes emerged in the conversation?"
- "How did the discussion evolve over time?"
- "What were the key turning points?"

**Comparative Questions:**
- "How did Twitter and Reddit discussions differ?"
- "What distinguished professor responses from student responses?"
- "Which stakeholder groups aligned in their views?"

## Exploring the Knowledge Graph

The knowledge graph contains all entities and relationships extracted from your seed materials, plus simulation-generated insights.

### Graph Search

```bash
POST /api/report/tools/search
Content-Type: application/json

{
  "graph_id": "mirofish_x1y2z3",
  "query": "student privacy concerns",
  "limit": 20
}
```

**Response:**

```json
{
  "success": true,
  "data": {
    "query": "student privacy concerns",
    "results_count": 18,
    "facts": [
      {
        "uuid": "fact_abc123",
        "fact": "Student activists organized a petition regarding AI privacy",
        "entities": ["Student Activists", "AI Privacy Petition"],
        "relevance_score": 0.92
      },
      {
        "uuid": "fact_def456",
        "fact": "University stated AI systems would not access student emails",
        "entities": ["State University", "AI Systems"],
        "relevance_score": 0.87
      }
    ]
  }
}
```

### Graph Statistics

```bash
POST /api/report/tools/statistics
{
  "graph_id": "mirofish_x1y2z3"
}
```

**Returns:**
- Total nodes and edges
- Entity type distribution
- Relationship type counts
- Graph density metrics

## Tips for Effective Interaction

### Interview Best Practices

<Steps>
  <Step title="Start Broad">
    Begin with open-ended questions: "What was your experience during the simulation?"
  </Step>
  
  <Step title="Drill Down">
    Follow up on interesting points: "You mentioned X, can you elaborate?"
  </Step>
  
  <Step title="Reference Specifics">
    Ask about particular actions: "Why did you post Y in round 23?"
  </Step>
  
  <Step title="Explore Relationships">
    Investigate interactions: "How did Z's comments influence you?"
  </Step>
</Steps>

## Advanced Exploration

### Cross-Platform Analysis

Compare agent behaviors across platforms:

```bash
# Twitter actions
GET /api/simulation/{simulation_id}/actions?platform=twitter&agent_id=23

# Reddit actions  
GET /api/simulation/{simulation_id}/actions?platform=reddit&agent_id=23
```

Agents often behave differently on Twitter (brief, reactive) vs. Reddit (detailed, discursive).

### Influential Agent Identification

Find agents who drove the conversation:

1. **High action count**: Many posts/comments
2. **High influence weight**: Configured as influential (from profiles)
3. **Viral content**: Posts that triggered many responses

```bash
GET /api/simulation/{simulation_id}/run/agent-stats
```

Sort by `total_actions` descending.

### Network Analysis

Identify clusters and communities:

```bash
POST /api/report/chat
{
  "simulation_id": "sim_m1n2o3p4",
  "message": "Which agents formed distinct communities or echo chambers?"
}
```

The Report Agent will use graph tools to identify interaction patterns.

## Exporting Data

### Download Action History

Action logs are stored as JSONL files in the simulation directory:

```
sim_m1n2o3p4/
├── twitter/actions.jsonl
└── reddit/actions.jsonl
```

Each line is a JSON object representing one action:

```json
{"round": 23, "agent_id": 17, "action_type": "CREATE_POST", "content": "...", "timestamp": "..."}
```

<Tip>
Use these files for custom analysis, visualization, or integration with external tools.
</Tip>

### Export Graph Data

```bash
GET /api/graph/data/{graph_id}
```

Download complete knowledge graph (nodes + edges) as JSON.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Agent gives generic responses">
    **Cause**: Agent may lack specific memories or context
    
    **Solution**: Ask about specific actions or timeframes:
    - Instead of "What do you think about X?"
    - Try "In round 25, you posted Y. What motivated that?"
  </Accordion>
  
  <Accordion title="Report Agent can't answer question">
    **Cause**: Question may be too vague or outside simulation scope
    
    **Solution**: 
    - Rephrase more specifically
    - Break complex questions into parts
    - Verify the information exists in the simulation
  </Accordion>
  
  <Accordion title="No action history for agent">
    **Cause**: Agent may not have been active during simulation
    
    **Check**:
    ```bash
    GET /api/simulation/{simulation_id}/profiles
    ```
    
    Verify agent exists and review their `activity_level` in config.
  </Accordion>
</AccordionGroup>

## Privacy and Ethics

<Warning>
**Important Reminder**: Agents are AI-generated personas, not real people. Their "opinions" and "experiences" are simulated based on:
- Seed material content
- Configured personas
- LLM model behaviors

Do not treat agent responses as statements from real individuals.
</Warning>

When using interview insights:
- Clearly label them as simulation-generated
- Cross-reference with actual data sources
- Use for scenario exploration, not as definitive predictions
- Consider biases in both source materials and AI models
