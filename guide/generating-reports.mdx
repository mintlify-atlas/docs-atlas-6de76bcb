## Overview

The Report Agent is an intelligent LLM-powered assistant that analyzes simulation results and generates detailed prediction reports. It uses ReACT (Reasoning + Acting) methodology to autonomously gather insights from the knowledge graph and simulation data, producing structured analysis documents.

## When to Generate Reports

<CardGroup cols={2}>
  <Card title="After Completion" icon="flag-checkered">
    **Best practice**: Wait for simulation to complete for full analysis
    
    All agent interactions and outcomes available
  </Card>
  
  <Card title="Mid-Simulation" icon="clock">
    **Interim analysis**: Generate reports during long-running simulations
    
    Provides preliminary insights and trends
  </Card>
</CardGroup>

## Starting Report Generation

```bash
POST /api/report/generate
Content-Type: application/json

{
  "simulation_id": "sim_m1n2o3p4",
  "force_regenerate": false
}
```

### Parameters

<ParamField path="simulation_id" type="string" required>
  The simulation ID to analyze
</ParamField>

<ParamField path="force_regenerate" type="boolean" default="false">
  Regenerate even if a report already exists
</ParamField>

### Response

```json
{
  "success": true,
  "data": {
    "simulation_id": "sim_m1n2o3p4",
    "report_id": "report_r1s2t3u4",
    "task_id": "task_v5w6x7y8",
    "status": "generating",
    "message": "Report generation started",
    "already_generated": false
  }
}
```

<Info>
The `report_id` is immediately returned, allowing you to monitor progress before completion.
</Info>

## Monitoring Report Progress

### Check Generation Status

```bash
POST /api/report/generate/status
Content-Type: application/json

{
  "task_id": "task_v5w6x7y8",
  "simulation_id": "sim_m1n2o3p4"
}
```

**Response:**

```json
{
  "success": true,
  "data": {
    "task_id": "task_v5w6x7y8",
    "status": "processing",
    "progress": 45,
    "message": "[Generating] Section 3/6: Key Findings",
    "created_at": "2026-03-13T15:30:00",
    "updated_at": "2026-03-13T15:33:15"
  }
}
```

### Real-time Section Progress

```bash
GET /api/report/{report_id}/sections
```

Monitor sections as they're generated:

```json
{
  "success": true,
  "data": {
    "report_id": "report_r1s2t3u4",
    "sections": [
      {
        "filename": "section_01.md",
        "section_index": 1,
        "content": "## Executive Summary\n\nThe simulation revealed..." 
      },
      {
        "filename": "section_02.md",
        "section_index": 2,
        "content": "## Simulation Background\n\nThis analysis..." 
      }
    ],
    "total_sections": 2,
    "is_complete": false
  }
}
```

<Tip>
Frontend applications can poll this endpoint to display report sections as they're written, providing a live generation experience.
</Tip>

## Report Content and Format

### Report Structure

The Report Agent generates a comprehensive document with:

<Steps>
  <Step title="Executive Summary">
    High-level overview of simulation findings and key predictions
  </Step>
  
  <Step title="Simulation Background">
    Context about the scenario, seed materials, and simulation setup
  </Step>
  
  <Step title="Key Findings">
    Major insights discovered through graph and simulation analysis
  </Step>
  
  <Step title="Agent Behaviors">
    Analysis of how different agent types responded and interacted
  </Step>
  
  <Step title="Predictions and Implications">
    Forward-looking analysis based on observed patterns
  </Step>
  
  <Step title="Methodology Notes">
    Transparency about data sources and analytical approaches
  </Step>
</Steps>

### Example Report Excerpt

```markdown
## Executive Summary

The simulation modeled public reaction to a university AI policy 
announcement across 72 hours of social media activity. Key findings:

1. **Initial Reception**: Mixed sentiment (42% supportive, 38% concerned, 
   20% neutral) with professors showing stronger support than students.

2. **Viral Dynamics**: 3 key posts drove 67% of engagement, with student 
   privacy concerns becoming the dominant discussion thread by hour 24.

3. **Stakeholder Alignment**: Faculty and administration showed aligned 
   messaging, while student groups fragmented into privacy advocates 
   vs. AI enthusiasts.

4. **Prediction**: Without addressing privacy concerns explicitly, 
   student opposition likely to intensify, particularly on Reddit 
   communities (projected 2.3x increase in critical posts).

## Key Findings

### Finding 1: Privacy Concerns Dominated Discussion

Graph analysis revealed "student privacy" emerged as the central topic, 
connecting to 47 distinct discussion threads...
```

### Download Report

```bash
GET /api/report/{report_id}/download
```

Downloads the full report as a `.md` file.

## Regenerating Reports

To create a new report with updated analysis:

```bash
POST /api/report/generate
{
  "simulation_id": "sim_m1n2o3p4",
  "force_regenerate": true
}
```

<Warning>
Regenerating creates a new report with a new `report_id`. The old report is preserved unless manually deleted.
</Warning>

## Report Quality Tips

### For Better Reports

<Steps>
  <Step title="Complete Simulations">
    Full simulations provide richer data for analysis
  </Step>
  
  <Step title="Clear Requirements">
    Specific simulation requirements lead to focused reports
  </Step>
  
  <Step title="Rich Seed Materials">
    More context in original documents = deeper insights
  </Step>
  
  <Step title="Diverse Agents">
    Multiple stakeholder types enable multi-perspective analysis
  </Step>
</Steps>

### Understanding Limitations

<Warning>
Reports are AI-generated analyses, not definitive predictions. They should inform decision-making, not replace human judgment.
</Warning>

**Considerations:**

- **Simulation Scope**: Reports reflect simulated data, not real-world certainty
- **LLM Hallucination**: Always verify critical claims against source data
- **Bias**: Agent personas and LLM models may introduce biases
- **Timeframe**: Short simulations may miss long-term dynamics

## Troubleshooting

<AccordionGroup>
  <Accordion title="Report generation failed">
    **Common causes:**
    - Insufficient simulation data (too few rounds/actions)
    - Graph query errors (check graph_id validity)
    - LLM API issues (rate limits, timeouts)
    
    **Check:**
    ```bash
    GET /api/report/{report_id}/console-log
    ```
  </Accordion>
  
  <Accordion title="Report is too generic">
    **Improve by:**
    - Making simulation requirements more specific
    - Running longer simulations for more data
    - Ensuring diverse agent behaviors in config
  </Accordion>
  
  <Accordion title="Missing expected insights">
    The Report Agent autonomously chooses what to analyze. If certain aspects are missing:
    - Use the **Interview** feature to query agents directly
    - Generate a new report with `force_regenerate`
    - Manually query the graph for specific information
  </Accordion>
</AccordionGroup>

## Deleting Reports

```bash
DELETE /api/report/{report_id}
```

Permanently removes the report and associated files.

<Info>
Deleting a report does not affect the underlying simulation or graph data.
</Info>
